# Changelog

Full release history. The README shows only the latest few entries.

## 1.13.x

- **1.13.0 — the four features from the dual review land.** (1) **Budget lease with session ownership**: `budget-open` records the opening session (`CLAUDE_CODE_SESSION_ID`) as `owner_sid`; a second session can no longer silently clobber another session's fresh open budget (explicit refusal, `--force` to override deliberately), and the SessionEnd reaper becomes ownership-aware — it closes its *own* budget immediately at any age, never touches a live foreign one (24h orphan horizon), keeping the 6h heuristic only for legacy budgets with no owner. (2) **Transcript-contract fixture suite** (`tests/transcript-contract/`): five static JSONL fixtures pin exactly which schema paths the enforcement trusts (plain usage, toolUseResult subagent aggregates, positional timestamp inference, cache fields with cache-reads excluded, alien-schema sentinel) and run through the REAL stop-hook and session-summary code — schema drift now fails at update time, not in production. (3) **Deterministic benchmark validators for shapes 01-03**: artifacts are saved per run/arm and `aggregate.py` recomputes expected outputs from the seeded fixtures (exact rows for 01-02, 0.05-tolerance means + SINTESI presence for 03, with the judgment half explicitly declared non-machine-gradeable); failed runs are now **reported per arm** instead of silently excluded. (4) **Versioned idempotent cache**: the effective cache key mixes plugin version and optional `--model` — a plugin upgrade or executor change invalidates stale hits by construction.

## 1.12.x

- **1.12.6 — concurrency and hot-path hardening, from the dual cross-family review (Codex + Gemini).** The four findings where both families converged, fixed: (1) **atomic state writes** everywhere (`tmp` + `os.replace`) — budget, delegations and statusline state files were bare read-modify-write: a partial write read by a concurrent hook was corrupt JSON, and the gate fails open on corrupt JSON = enforcement silently off; (2) **SQLite configured for concurrency** (WAL + `busy_timeout` in every writer and in the statusline query) — under lock contention `gate_deny`/`read_dedup`/`effort_mismatch` events were silently dropped, presenting incomplete telemetry as observed behavior; (3) **`session_id` sanitized** (strict `[A-Za-z0-9_-]{1,64}` allowlist, reject-not-normalize) before entering any state path — a transcript-supplied sid with separators could escape the state directories; (4) **the Stop hook goes incremental**: it re-parsed the whole transcript JSONL at every turn end (quadratic on long budgeted tasks) — now a state file next to the budget keeps byte offset, cumulative totals and sentinel counters, keyed on `declared_at`, reading only new complete lines (a partial trailing line is picked up next turn: a live indicator may drop it, enforcement may not). Verified: incremental totals match full-rescan semantics through 2×-warn and 3×-block, WAL active, hostile sid writes nothing, zero leftover tmp files. Feature proposals from the same review (budget leases, transcript fixture suite, benchmark validators, cache versioning) tracked separately.


- **1.12.5 — Windows: enforcement was a silent no-op (issue #1, first external report).** Two POSIX-only assumptions, reported with repros by @AlvaroRojas and both confirmed: (1) hook output with non-Latin-1 characters (`≈ → × ▶`) crashed on the cp1252 codepage and the gate's fail-open swallowed the crash — **every un-budgeted delegation was silently allowed on Windows**; all five Python entry points now reconfigure stdout/stderr to UTF-8 (harmless on POSIX). (2) `cwd_slug` kept backslashes and the drive colon, so the budget file could never be written on Windows (`OSError`), and gate (forward-slash cwd from the hook payload) vs telemetry (backslash `os.getcwd()`) produced different slugs — the slug base is now OS-agnostic (`re.sub` on non-alphanumerics, as the reporter suggested) and the anti-collision hash runs on the canonicalized path (`\\`→`/`), so both spellings collapse to one slug. Aligned across all seven consumers — including `session-cost-report.py`, which the 1.12.4 slug change had missed. Open budgets in the old formats get migrated by name recomputation from their stored `cwd`.


- **1.12.4 — five hardening fixes from the cross-family review.** An adversarial review by Codex (gpt-5.6-terra, read-only) found real gaps; all confirmed against the code and fixed: (1) `budget-open` now rejects non-positive/non-integer estimates — `--expected-output 0` used to silently disable the whole 2×/3× enforcement; (2) the cwd slug gains a hash suffix (`a.b` and `a-b` used to collide on the same budget file), kept identical across its five consumers (telemetry, gate, Stop hook, statusline, bench harness); (3) the SessionEnd budget reaper only closes budgets older than 6h — it used to silently kill the enforcement of a *concurrent live session* on the same cwd; (4) input truncation on the external scripts is now fail-closed: oversize input → explicit error, truncation only via explicit `--allow-truncate` (a verdict or deliverable computed on a silently cut input is the worst outcome); (5) `external-exec.py` now verifies an open budget itself — it runs via Bash, which the PreToolUse gate never sees, so the end-to-end enforcement claim is now actually true. Plus: budget input accounting explicitly documented as a *fresh-token* budget (cache reads excluded by design) in hook, skill and messages. Findings 6-7 (README wording) tracked separately with the reviewer's rewrite proposals.

- **1.12.3 — DeepSeek lane removed; cross-family goes two-lane.** The last free DeepSeek variant left OpenRouter (verified live against the models API: zero `deepseek/*:free` entries remain), so every `--provider deepseek` call would fail. Removed from defaults and docs rather than replaced: two uncorrelated families (Gemini + GPT/Codex) are enough for ladder rung 4, and a lane that can silently die isn't worth its maintenance. Grok evaluated as a replacement and rejected: no durable API free tier (the consumer app tier has no API), and the credits-for-data-sharing program is both unreliable and unacceptable for client work.
- **1.12.2 — cross-family defaults upgraded, verified live.** `gemini-3-flash-preview` (same free-tier limits, intermittent 503 under GA rush — unavailable stays loud) and `gpt-5.6-terra` (GA'd 2026-07-09, needs Codex CLI ≥0.144). New explicit `gemini-stable` provider (2.5-flash) as a *named* fallback — fallback by explicit `--provider` choice, never silent. `gemini-3.5-flash` exists but 503s consistently on the free tier at launch; noted for a later retry.
- **1.12.1 — fast path + positioning, both born from the benchmark.** The measured small-task overhead had a behavioral share (policy ceremony on tasks too small to benefit): the kernel now carries an explicit **fast path** — a task that fits in a single turn with no delegation → just execute, zero budget, zero ritual; *the policy must never cost more than the task it governs*. What is NOT done: weakening axis 2 or verification to make the small-task number look better — that would be the exact Goodhart failure the kernel forbids; the fixed kernel share stays and is documented as an insurance premium. README gains the three-clause positioning (quality as a constraint / deterministic enforcement via blocking scripts, not prompts / transparency via telemetry and benchmarks that publish their own limits) and a plain-language "What the benchmarks actually say" section — negative findings included.

- **1.12.0 — external executor (experimental): non-code batches off Claude quota.** New `scripts/external-exec.py`: routes axis-4 non-code items (extraction, classification, text transform) to free external tiers — Gemini Flash (~1,500 req/day free API) or Codex CLI — at **zero Claude tokens**. Reuses the cross-family config and discipline: no silent fallback (`unavailable` → run the normal Claude route), executor contract in the system prompt (spec verbatim, deliverable only, honest `NEEDS_CONTEXT` → exit 2), built-in rung-1 (`--schema-json` refuses to hand malformed JSON downstream), the wrapper is the only writer (CLI providers stay read-only). Every run logs `external_exec` `{provider, type, ok, check}` and `report` breaks ok-rate down per provider/type with the same density rule as everything else: the route stays **per-case and experimental until the data is DENSE (N≥10)** — model quality is measured, never asserted. Verified live: Gemini Flash classified a 3-review sentiment batch 3/3 with valid JSON on the first call. Pre-budget still owed (`--route script --effort low` is the honest declaration).

## 1.11.x

- **1.11.1 — `[BDG]` shows the bust coming, not fired.** The statusline segment upgrades from state (`ok/2×/3×`) to a **live ratio**: `[BDG 0.7×·high]` = output consumed so far vs the declared estimate, plus the budget's declared effort tier. Same accounting as the Stop hook (recursive `usage` walk, records after `declared_at`, positional timestamp inference) but **incremental** — the counter lives in the existing per-session state file next to the `[DLG]` offset, so each render reads only new transcript lines, never a rescan; it resets when `declared_at` changes. Colors mirror the enforcement thresholds (green < 2×, yellow ≥ 2×, red ≥ 3×): the 2× checkpoint becomes visible on approach instead of arriving as a surprise block. Without a transcript it degrades to the old budget-file-only badge (`ok/2×`, effort suffix kept). One trap worth documenting: the whole metric pass lives in a single-quoted shell string — an apostrophe in an Italian comment (`all'offset`) silently truncated it; comments in that block are now apostrophe-free by rule.
- **1.11.0 — effort becomes a routing lever (pinned agents + declared tier).** The Agent tool has no per-call effort parameter, so the only real way to control reasoning effort on delegations is an agent definition with `effort` pinned in frontmatter. The plugin now ships two: **`fd-executor`** (sonnet, `effort: low` — the axis-4 batch executor: spec-verbatim, forced schema, status token, 3-failure cap, explicit permission to ABSTAIN) and **`fd-verifier`** (inherit, `effort: high` — the rung-3 adversarial verifier: read-only, artifact+rubric only, one CONFIRMED/REFUTED/UNVERIFIABLE verdict per finding). `budget-open` gains an optional **`--effort low|medium|high|xhigh|max`** (validated, stored in the budget file): a decision record, not a selection constraint. The gate checks **coherence** between declared and pinned tier — on mismatch it prints a one-line warn and logs an `effort_mismatch` event, **never denies** (blocking on a declared budget field would be the exact Goodhart failure the kernel forbids). `report` gains the per-tier breakdown (tasks and flag-rate per declared effort, mismatch pairs): the data that will decide any future warn→deny promotion, per the plugin's own rule — enforcement follows measurement, never precedes it. Older Claude Code versions ignore the `effort` frontmatter field: agents still work, they inherit the session effort (silent degradation, documented in Known limits).

## 1.10.x

- **1.10.10 — self-review: 6 real bugs found and fixed by the plugin's own review ladder.** A high-effort code review of the whole 1.10.x line (8 finder angles → inline verify → cross-family check on both Gemini *and* Codex lanes) surfaced and fixed: the cost-checkpoint `ask` path skipped `record_delegation()`, so exactly the priciest delegations vanished from `[DLG]` (now recorded before the checkpoint); `git_yield` passed a naive `--since` to git, shifting the commit window by the host's UTC offset (now `%z`); `session-cost-report` double-counted every transcript when `CLAUDE_CONFIG_DIR` points at `~/.claude` (bases now deduped); cross-verify discarded valid verdicts fenced as ```` ```JSON ```` (parsing now case-insensitive); the statusline `≡` marker never matched versioned model ids or "Claude X" display names (normalizer now strips prefix and date suffix); a missing `declared_at` was misreported as a stale budget (now an honest "corrupt file" deny). Plus two hot-path efficiency cuts — the Stop hook no longer re-walks the whole transcript history every turn (sentinel-only records are skipped once satisfied) and `events(event, ts)` gets an index for the `[XF]` statusline query — and doc drift fixes (gate matcher is `Agent|Task|Workflow` everywhere; benchmarks README now says 4 shapes / 24 sessions). Methodological note: on findings whose premise lay outside the shown snippet, Codex answered `uncertain` where Gemini said `refuted` — same substance, better-calibrated label; both lanes confirmed every mechanism, none refuted a finding.
- **1.10.9 — cost checkpoint: an expensive task is your call, not a silent spend.** When a task genuinely needs the top model (axis 2) *and* is expensive, you decide whether to spend the tokens — the plugin no longer just burns them. For **delegated** work it's enforced: if the pre-budget's `expected-output` exceeds a ceiling (default 50k, `FD_COST_CEILING` or `cost-checkpoint.json` to tune), the gate returns `ask` instead of `allow`, surfacing the estimate and your remaining weekly quota — approve and it proceeds, or reopen the budget with `--cost-ack` to stop re-asking on that task. The statusline now bridges your live quota to the gate (`quota.json`), so when the weekly limit runs low the ceiling drops automatically (a mid-size task becomes worth confirming). For **inline** work — which no hook can intercept — the kernel instructs the top model to present the choice itself: estimate, % of limit left, why the top model is needed, and the alternatives (split / cheap-executor+verify / defer to reset). It's the interactive, context-aware cousin of a hard dollar cap: not a blind stop, an informed decision that respects the rate limit as *your* scarce resource. Fail-open throughout.
- **1.10.8 — the top model owns "done".** Before delegating, the kernel now requires a **verifiable done** (a command that passes / an observable behavior / an enumerable checklist) plus a **stop condition** with a failure cap — and if done isn't verifiable, you don't delegate until it is. Executors never self-assess completion. This is the up-front twin of the verification ladder (which is *how* you check; this is *what* counts as finished) and closes the one gap peers like [helm](https://github.com/Jacknelson6/helm) and the 10-80-10 routing writeups all point at. It's the plan-half of "plan big, execute small": a cheap executor self-declaring success is exactly the failure this prevents.
- **1.10.7 — the two silent things now show up in `report`.** (1) **read-dedup savings are measured, not silent:** every suppressed re-read logs a `read_dedup` event with the tokens saved (≈ chars/4), and `report` sums them — so enabling the opt-in hook produces a real number to validate against, not a vibe. (2) **Yield analysis (token per commit):** at `SessionEnd`, if the cwd is a git repo, the summary logs how many commits landed during the session window, and `report` prints output-tokens-per-commit. It measures *return* — cost tells you what you spent, this tells you whether the spend produced kept work. Strictly a **diagnostic alarm, never a target**: planning/debug/review sessions legitimately don't commit, so a high (or zero) ratio flags where to look, it never condemns.
- **1.10.6 — cross-family verifier is now data-driven by task type.** Rung 4 (the cross-family check) gains a `--type` slug that the telemetry logs, and `report` breaks the cross-family hit-rate down **per task type**. This answers *which task types cross-family actually catches errors on* empirically, instead of asserting model-family strengths (a claim that decays every release). Seed candidates in the skill — `cross-lingua` (IT↔EN deliverables: a different family's multilingual training is genuinely uncorrelated), `security-review`, `spec-compliance` — are a starting hypothesis confirmed or killed by the logged data (same `N≥10` discipline as the per-type budget density table), never doctrine. It stays a **skill-level suggestion**, not an auto-hook: auto-firing on a fuzzy "is this type X?" classification would burn free-tier quota.
- **1.10.5 — lossless read-dedup (opt-in) + token-reduction stance.** New `read-dedup.py` PostToolUse hook: on a **re-read** of a file already seen this session it replaces Read's output with just the diff (or an "unchanged" marker), cutting the biggest agentic token sink — re-sending file content already in context. **Lossless and recoverable:** it only touches large files, skips partial (offset/limit) reads, and after a dedup the *next* read of that file passes through in full — so the model always has a one-read path back to full content even after a compact. **Opt-in by design** (not wired into the default hooks — colleagues on the marketplace build don't inherit it until you validate it): enable with `export FD_READ_DEDUP=1` or `touch ~/.claude/fable-director/read-dedup.on`, then add the hook (see [Token reduction](#-token-reduction-lossless-only)). The routing engine optimizes **cost per token**; this is the first lever that cuts the **token count** itself — and by policy it stays **lossless-only** (RAG-style lossy chunking is now a documented anti-pattern: −90% tokens bought with a correctness risk is the exact Goodhart failure the kernel forbids).
- **1.10.4 — gate denials become telemetry.** Every deny from the pre-delegation gate now logs a `gate_deny` event (kind `no_budget` / `stale_budget` / `flagged`, plus tool and declared model) and `report` prints the breakdown. Without it, post-hoc analysis couldn't distinguish "never attempted delegation" from "denied and fell back inline" — the exact blind spot hit while reading the shape-04 bench. Also fixed a systematically ambiguous shape-04 fixture: a quality-negative phrase ("pannello che si stacca") both arms read as a safety hazard against a ground truth of NO — safety-precision from earlier runs is not comparable (see [benchmarks/README](benchmarks/README.md)).
- **1.10.2 — cross-family lanes verified live.** Default Gemini model for new configs is `gemini-2.5-flash` (verified with a real call: `gemini-3-flash` doesn't exist on the AI Studio free tier, `gemini-flash-latest` answers 503 under load). Both lanes tested end-to-end — Gemini free API and Codex CLI (ChatGPT login) each returned a correct adversarial verdict. **When they fire and how to invoke them yourself: see [Cross-family verifier](#-cross-family-verifier--when-and-how) below.**
- **1.10.1 — `[DLG]` shows effective tokens, not declared calls.** The segment now reads the session transcript **incrementally** (a state file keeps the byte offset — each refresh parses only new lines, never a rescan) and shows **output tokens per effective model** of subagent work: `[DLG SONNET-5 41k HAIKU-4-5 3k]` (`≡` = subagents inheriting the main-loop model). Immune to the quiet-fallback blind spot and it weighs work, not call counts. Where the transcript isn't exposed it degrades to the gate's declared-call registry, marked `≈` — the two modes are visually distinct by design. Also new: **`[XF]` cross-family segment** — `GEMINI▲` while a `cross-verify.py` call is in flight, `CODEX×2` for today's calls (local telemetry; free tiers expose no quota API, so this is presence, not remaining quota).
- **1.10.0 — `[DLG]` statusline segment (declared calls).** The pre-delegation gate keeps a per-session registry of delegations counted by declared model. The registry dies at SessionEnd (48h orphan cleanup for crashed sessions).
- **Benchmark task shape 04 — where governance actually bites.** 40 synthetic customer reviews requiring per-item semantic judgment (closed-vocabulary sentiment/theme + safety-defect flagging, 6 planted defects with ground truth outside the task's view). Unlike shapes 01–03 (script-parity), this one can trigger real delegation → the gate and Stop hook finally get exercised in the `on` arm. `aggregate.py` now also reports **accuracy per arm** (sentiment/theme accuracy, safety recall/precision): savings only count at verified-equal quality — safety recall lost in one arm gets reported, not hidden.

## 1.9.x

- **1.9.2 — in-session model visibility.** When a delegation declares an explicit model (`Agent` calls with a `model` field), the gate now prints a one-line notice in session: `FD ▶ delega a modello esplicito: <agent> → <model>`. Inherit stays silent (homogeneous fan-outs produce zero noise). The line shows the *declared* model — the effective one can quietly degrade (see Known limits); post-task truth is `session-cost-report.py` (per-model token breakdown, already shipped).
- **1.9.1 — Codex CLI provider + local usage counter.** Third cross-family lane: `"type": "cli"` providers run a subprocess (Codex CLI with ChatGPT login — fixed-cost, no API billing); spec via stdin, unique mktemp output, `command -v` preflight, quota errors → `unavailable`. New `cross-verify.py --usage`: free tiers expose no quota API (neither Gemini nor ChatGPT/Codex), so this counts today's calls per provider from local telemetry against the declared limits in config (honest label: local counter, blind to key usage elsewhere; a real 429 still fails loudly).
- **Cross-family verifier (optional, ladder rung 4).** `scripts/cross-verify.py` — adversarial check on highest-stakes claims by a **different model family** (all-Claude ensembles share correlated blind spots by construction), out of Claude quota. Zero dependencies (stdlib HTTP), OpenAI-compatible endpoints behind a config file (`--init` creates it: Gemini free API / DeepSeek via OpenRouter free — URLs and models live in config, not code, because free tiers change monthly). **No silent fallback**: missing key, rate limit, endpoint down → `STATUS: unavailable` with the explicit instruction to degrade to the same-family fresh-context verifier — unavailable is never "verified".
- **Benchmark now measures the shipped stack.** The `on` arm injects the full enforcement stack via `--settings` (SessionStart kernel + PreToolUse gate + Stop 2×/3× check) instead of kernel-only `--append-system-prompt`. Budget files are wiped between runs. The published number, when it lands, will measure what you actually install.

## 1.8.x

- **1.8.2 — fresh-context verifier + known limits.** Verification ladder rung 3 now requires the LLM verifier to run in a fresh-context subagent that sees only artifact + rubric — never the maker's reasoning trail (inline self-critique is structurally self-preferential). New "Known limits" README section: enforcement is local-only (cloud/CMA sessions run outside the hook stack), statusline degradation, Claude Code's quiet model fallback, transcript schema dependency.
- **1.8.1 — skill-audit fixes.** "Never delegate" lists aligned between kernel and skill (the kernel was missing "decisions on how to count or report"); the skill's trigger description now covers gate denials; new playbook seed: spot delegations get an honest micro-budget for the single call — never a wide session-budget that kills the 2×/3× thresholds.
- **Cache-staleness sentinel.** Claude Code copies plugins to a cache at install time and never re-checks it: with a local marketplace the running plugin silently falls behind its source (we lived it: 1.0.0 running for days while the source was at 1.6.0). A `SessionStart` sentinel now compares the running version against every `directory`-type marketplace source and warns with the exact `claude plugin update` command. Warn only, never auto-update — a nested CLI call in a hook is slow, races the plugin registry, and the current session would stay on the old version anyway.
- **`/fable-director:review`** — the director reads its own telemetry. The learning loop used to write heuristics only on incidents (3× busts, rule-of-3); this command has the top model read `report` + playbook and produce a brutally honest improvement plan anchored **only to objective alarms** (max 5 recommendations, each citing the datum that justifies it; "no intervention justified by the data" is a valid outcome — inventing problems is forbidden).

## 1.7.0

- **Pre-delegation gate (the bootstrap gap is closed).** Until 1.6.0 the 2×/3× enforcement only bit *if* a budget had been opened — and opening it was a prompt-level instruction the model could skip. A new `PreToolUse` hook now **denies any `Agent`/`Task`/`Workflow` call with no open budget** for the cwd, replying with the exact `budget-open` command to run. Also denies delegation while a budget is `flagged` (no dodging the 3× post-mortem by delegating) or older than 24h. Fail-open by design: a gate bug can never block a legitimate delegation.
- **Transcript schema sentinel.** Token accounting reads Claude Code's undocumented JSONL transcript format; a silent field rename used to zero the counters without any error. Both the `Stop` hook and `session-summary` now detect "many valid records, zero recognized `usage`/`timestamp`" and **fail loudly**: one-time warning, `schema_anomaly` telemetry event, enforcement suspended instead of trusting zeros.
- **5-part spec contract for delegation prompts** (Objective / Files / Interfaces / Constraints / Verification) — replaces the old 4-component contract; context-free delegation is the test that the route is delegable at all.
- Statusline example renamed `[OPUS4.8]` → `[FABLE5]` (the director role matches the plugin's name; the model was never hardcoded).
