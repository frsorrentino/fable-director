# Changelog

Full release history. The README shows only the latest few entries.

## 1.14.x

- **1.14.0 ‚Äî write perimeter, task receipts, MCP metering.** Three new capabilities from the new-territory review, all at ~zero model-token cost because they live in hooks and scripts. (1) **Write perimeter** (new `PreToolUse` hook on `Write`/`Edit`/`NotebookEdit`): the budget can declare WHERE the task may write (`budget-open --paths "src/parser/*,tests/*"`) ‚Äî in-project writes outside it are denied with the exact amendment command (`budget-amend --add-paths ... --reason ...`, explicit and logged; files outside the project such as scratchpads are never constrained). Independently, user-owned `never_write` patterns (`.fd-perimeter.json` in the project or `~/.claude/fable-director/perimeter.json`) are denied unconditionally ‚Äî the kernel's "production writes without prior backup" turned from advice into a wall. The budget bounded what a task may SPEND; now it also bounds what it may TOUCH. (2) **Task receipts**: every `budget-close` writes a local machine-readable receipt (estimate vs actual, verify contract, data class, perimeter + amendments, outcome, plugin version) under `receipts/`, capped at 200 ‚Äî provenance for audit, written by script, zero tokens. (3) **MCP metering** (new `PostToolUse` hook on `mcp__*`): logs bytes each MCP server pushes into context; `report` aggregates per server ‚Äî the 2-20k silent-bloat band finally measured. `report` also counts perimeter denies/amendments (many amendments = your declared perimeters are systematically too narrow ‚Äî a calibration signal, like the estimates).

(2) **7-day burn-rate**: the statusline appends a compact quota history (write-on-change, capped) and `/status` projects "at this pace, 100% around Thu 18:00" from the monotonic tail since the last reset ‚Äî minimum 3 samples/3h/0.5% growth, otherwise silence, never an invented projection. (3) **Script-promotion queue**: `report` lists task types recurring ‚â•2 closed-ok on model routes with the tokens spent ‚Äî the evidence for crystallization; the decision stays human. (4) **Quality-evidence contract**: `budget-open --verify "cmd/checklist"` makes the kernel's verifiable-done machine-readable; the gate warns once per budget when absent (never denies ‚Äî exploratory work keeps its freedom). (5) **Enforceable privacy boundary**: `--data-class restricted` on the budget makes `external-exec.py` AND `cross-verify.py` refuse deterministically ‚Äî the README's privacy disclosure becomes script-side enforcement. - **1.13.3 ‚Äî self-audit of the instruction files (the section-5 audit from the field).** The plugin's own instruction files, audited with the lens "rules written for weaker models keep the new model behaving like a weaker model": (1) the hardcoded "Multi-agent ‚âà 15√ó chat cost" multiplier (uncited, from another model era) becomes "an order of magnitude ‚Äî measure YOUR ratio via report delegation_overhead"; (2) the idempotent-cache key spec caught up with 1.13.0 reality (plugin version + `--model` mixed into the effective key); (3) `/fable-director:review` now also reads the three telemetry signals added since 1.11 (per-tier effort flag-rate, `effort_mismatch`, external-executor ok-rate) ‚Äî it was reviewing yesterday's data model; (4) the kernel's enforcement sentence now includes `external-exec.py` self-enforcement. Reviewed and deliberately KEPT: no-progress termination (cheap safety net at any model tier), canary + never-trust-success-report + 5-part spec (they govern the mid-tier executors, not the orchestrator ‚Äî and the shape-05 bench just measured that this discipline is exactly where the value lives). - **1.13.2 ‚Äî `/fable-director:status`: the statusline for clients that have no statusline.** Remote-control clients (smartphone, web) don't render the terminal statusline. The new command prints the same segments as plain conversation text, rebuilt from the on-disk state files ‚Äî with honest freshness: budget ratio/effort and delegations are always current (hooks write them every turn), plan quotas are labeled *as-of the last statusline render* with their age, and degrade to "n/d" instead of pretending. Zero model tokens to compute (one script), works on any client.

- **1.13.1 ‚Äî dual-account: one shared memory, per-account quotas.** With two Claude Code accounts (the default one plus any `CLAUDE_CONFIG_DIR`-based second profile) the learning memory was already shared **by design** ‚Äî playbook, telemetry, budgets and counters are anchored to `$HOME`, not to the account's config dir, so a lesson learned on one account guides the other. What was NOT correct: `quota.json` (the statusline‚Üígate bridge for the cost checkpoint) was a single shared file, so the gate could lower its thresholds based on the *other* account's plan quotas. Now the statusline writes `quota-<account-hash>.json` and the gate reads the file matching its own `CLAUDE_CONFIG_DIR` (legacy `quota.json` fallback kept). `session_summary` events also gain an `account` field (config-dir basename) so `report` can tell the two experiences apart while learning from both. Verified end-to-end: same budget, scarce-quota account ‚Üí `ask`, abundant-quota account ‚Üí allow.

- **1.13.0 ‚Äî the four features from the dual review land.** (1) **Budget lease with session ownership**: `budget-open` records the opening session (`CLAUDE_CODE_SESSION_ID`) as `owner_sid`; a second session can no longer silently clobber another session's fresh open budget (explicit refusal, `--force` to override deliberately), and the SessionEnd reaper becomes ownership-aware ‚Äî it closes its *own* budget immediately at any age, never touches a live foreign one (24h orphan horizon), keeping the 6h heuristic only for legacy budgets with no owner. (2) **Transcript-contract fixture suite** (`tests/transcript-contract/`): five static JSONL fixtures pin exactly which schema paths the enforcement trusts (plain usage, toolUseResult subagent aggregates, positional timestamp inference, cache fields with cache-reads excluded, alien-schema sentinel) and run through the REAL stop-hook and session-summary code ‚Äî schema drift now fails at update time, not in production. (3) **Deterministic benchmark validators for shapes 01-03**: artifacts are saved per run/arm and `aggregate.py` recomputes expected outputs from the seeded fixtures (exact rows for 01-02, 0.05-tolerance means + SINTESI presence for 03, with the judgment half explicitly declared non-machine-gradeable); failed runs are now **reported per arm** instead of silently excluded. (4) **Versioned idempotent cache**: the effective cache key mixes plugin version and optional `--model` ‚Äî a plugin upgrade or executor change invalidates stale hits by construction.

## 1.12.x

- **1.12.6 ‚Äî concurrency and hot-path hardening, from the dual cross-family review (Codex + Gemini).** The four findings where both families converged, fixed: (1) **atomic state writes** everywhere (`tmp` + `os.replace`) ‚Äî budget, delegations and statusline state files were bare read-modify-write: a partial write read by a concurrent hook was corrupt JSON, and the gate fails open on corrupt JSON = enforcement silently off; (2) **SQLite configured for concurrency** (WAL + `busy_timeout` in every writer and in the statusline query) ‚Äî under lock contention `gate_deny`/`read_dedup`/`effort_mismatch` events were silently dropped, presenting incomplete telemetry as observed behavior; (3) **`session_id` sanitized** (strict `[A-Za-z0-9_-]{1,64}` allowlist, reject-not-normalize) before entering any state path ‚Äî a transcript-supplied sid with separators could escape the state directories; (4) **the Stop hook goes incremental**: it re-parsed the whole transcript JSONL at every turn end (quadratic on long budgeted tasks) ‚Äî now a state file next to the budget keeps byte offset, cumulative totals and sentinel counters, keyed on `declared_at`, reading only new complete lines (a partial trailing line is picked up next turn: a live indicator may drop it, enforcement may not). Verified: incremental totals match full-rescan semantics through 2√ó-warn and 3√ó-block, WAL active, hostile sid writes nothing, zero leftover tmp files. Feature proposals from the same review (budget leases, transcript fixture suite, benchmark validators, cache versioning) tracked separately.


- **1.12.5 ‚Äî Windows: enforcement was a silent no-op (issue #1, first external report).** Two POSIX-only assumptions, reported with repros by @AlvaroRojas and both confirmed: (1) hook output with non-Latin-1 characters (`‚âà ‚Üí √ó ‚ñ∂`) crashed on the cp1252 codepage and the gate's fail-open swallowed the crash ‚Äî **every un-budgeted delegation was silently allowed on Windows**; all five Python entry points now reconfigure stdout/stderr to UTF-8 (harmless on POSIX). (2) `cwd_slug` kept backslashes and the drive colon, so the budget file could never be written on Windows (`OSError`), and gate (forward-slash cwd from the hook payload) vs telemetry (backslash `os.getcwd()`) produced different slugs ‚Äî the slug base is now OS-agnostic (`re.sub` on non-alphanumerics, as the reporter suggested) and the anti-collision hash runs on the canonicalized path (`\\`‚Üí`/`), so both spellings collapse to one slug. Aligned across all seven consumers ‚Äî including `session-cost-report.py`, which the 1.12.4 slug change had missed. Open budgets in the old formats get migrated by name recomputation from their stored `cwd`.


- **1.12.4 ‚Äî five hardening fixes from the cross-family review.** An adversarial review by Codex (gpt-5.6-terra, read-only) found real gaps; all confirmed against the code and fixed: (1) `budget-open` now rejects non-positive/non-integer estimates ‚Äî `--expected-output 0` used to silently disable the whole 2√ó/3√ó enforcement; (2) the cwd slug gains a hash suffix (`a.b` and `a-b` used to collide on the same budget file), kept identical across its five consumers (telemetry, gate, Stop hook, statusline, bench harness); (3) the SessionEnd budget reaper only closes budgets older than 6h ‚Äî it used to silently kill the enforcement of a *concurrent live session* on the same cwd; (4) input truncation on the external scripts is now fail-closed: oversize input ‚Üí explicit error, truncation only via explicit `--allow-truncate` (a verdict or deliverable computed on a silently cut input is the worst outcome); (5) `external-exec.py` now verifies an open budget itself ‚Äî it runs via Bash, which the PreToolUse gate never sees, so the end-to-end enforcement claim is now actually true. Plus: budget input accounting explicitly documented as a *fresh-token* budget (cache reads excluded by design) in hook, skill and messages. Findings 6-7 (README wording) tracked separately with the reviewer's rewrite proposals.

- **1.12.3 ‚Äî DeepSeek lane removed; cross-family goes two-lane.** The last free DeepSeek variant left OpenRouter (verified live against the models API: zero `deepseek/*:free` entries remain), so every `--provider deepseek` call would fail. Removed from defaults and docs rather than replaced: two uncorrelated families (Gemini + GPT/Codex) are enough for ladder rung 4, and a lane that can silently die isn't worth its maintenance. Grok evaluated as a replacement and rejected: no durable API free tier (the consumer app tier has no API), and the credits-for-data-sharing program is both unreliable and unacceptable for client work.
- **1.12.2 ‚Äî cross-family defaults upgraded, verified live.** `gemini-3-flash-preview` (same free-tier limits, intermittent 503 under GA rush ‚Äî unavailable stays loud) and `gpt-5.6-terra` (GA'd 2026-07-09, needs Codex CLI ‚â•0.144). New explicit `gemini-stable` provider (2.5-flash) as a *named* fallback ‚Äî fallback by explicit `--provider` choice, never silent. `gemini-3.5-flash` exists but 503s consistently on the free tier at launch; noted for a later retry.
- **1.12.1 ‚Äî fast path + positioning, both born from the benchmark.** The measured small-task overhead had a behavioral share (policy ceremony on tasks too small to benefit): the kernel now carries an explicit **fast path** ‚Äî a task that fits in a single turn with no delegation ‚Üí just execute, zero budget, zero ritual; *the policy must never cost more than the task it governs*. What is NOT done: weakening axis 2 or verification to make the small-task number look better ‚Äî that would be the exact Goodhart failure the kernel forbids; the fixed kernel share stays and is documented as an insurance premium. README gains the three-clause positioning (quality as a constraint / deterministic enforcement via blocking scripts, not prompts / transparency via telemetry and benchmarks that publish their own limits) and a plain-language "What the benchmarks actually say" section ‚Äî negative findings included.

- **1.12.0 ‚Äî external executor (experimental): non-code batches off Claude quota.** New `scripts/external-exec.py`: routes axis-4 non-code items (extraction, classification, text transform) to free external tiers ‚Äî Gemini Flash (~1,500 req/day free API) or Codex CLI ‚Äî at **zero Claude tokens**. Verified live: Gemini Flash classified a 3-review sentiment batch 3/3 with valid JSON on the first call. Pre-budget still owed (`--route script --effort low` is the honest declaration).

## 1.11.x

- **1.11.1 ‚Äî `[BDG]` shows the bust coming, not fired.** The statusline segment upgrades from state (`ok/2√ó/3√ó`) to a **live ratio**: `[BDG 0.7√ó¬∑high]` = output consumed so far vs the declared estimate, plus the budget's declared effort tier. Same accounting as the Stop hook (recursive `usage` walk, records after `declared_at`, positional timestamp inference) but **incremental** ‚Äî the counter lives in the existing per-session state file next to the `[DLG]` offset, so each render reads only new transcript lines, never a rescan; it resets when `declared_at` changes. Colors mirror the enforcement thresholds (green < 2√ó, yellow ‚â• 2√ó, red ‚â• 3√ó): the 2√ó checkpoint becomes visible on approach instead of arriving as a surprise block. Without a transcript it degrades to the old budget-file-only badge (`ok/2√ó`, effort suffix kept). One trap worth documenting: the whole metric pass lives in a single-quoted shell string ‚Äî an apostrophe in an Italian comment (`all'offset`) silently truncated it; comments in that block are now apostrophe-free by rule.
- **1.11.0 ‚Äî effort becomes a routing lever (pinned agents + declared tier).** The Agent tool has no per-call effort parameter, so the only real way to control reasoning effort on delegations is an agent definition with `effort` pinned in frontmatter. The plugin now ships two: **`fd-executor`** (sonnet, `effort: low` ‚Äî the axis-4 batch executor: spec-verbatim, forced schema, status token, 3-failure cap, explicit permission to ABSTAIN) and **`fd-verifier`** (inherit, `effort: high` ‚Äî the rung-3 adversarial verifier: read-only, artifact+rubric only, one CONFIRMED/REFUTED/UNVERIFIABLE verdict per finding). `budget-open` gains an optional **`--effort low|medium|high|xhigh|max`** (validated, stored in the budget file): a decision record, not a selection constraint. The gate checks **coherence** between declared and pinned tier ‚Äî on mismatch it prints a one-line warn and logs an `effort_mismatch` event, **never denies** (blocking on a declared budget field would be the exact Goodhart failure the kernel forbids). `report` gains the per-tier breakdown (tasks and flag-rate per declared effort, mismatch pairs): the data that will decide any future warn‚Üídeny promotion, per the plugin's own rule ‚Äî enforcement follows measurement, never precedes it. Older Claude Code versions ignore the `effort` frontmatter field: agents still work, they inherit the session effort (silent degradation, documented in Known limits).

## 1.10.x

- **1.10.10 ‚Äî self-review: 6 real bugs found and fixed by the plugin's own review ladder.** A high-effort code review of the whole 1.10.x line (8 finder angles ‚Üí inline verify ‚Üí cross-family check on both Gemini *and* Codex lanes) surfaced and fixed: the cost-checkpoint `ask` path skipped `record_delegation()`, so exactly the priciest delegations vanished from `[DLG]` (now recorded before the checkpoint); `git_yield` passed a naive `--since` to git, shifting the commit window by the host's UTC offset (now `%z`); `session-cost-report` double-counted every transcript when `CLAUDE_CONFIG_DIR` points at `~/.claude` (bases now deduped); cross-verify discarded valid verdicts fenced as ```` ```JSON ```` (parsing now case-insensitive); the statusline `‚â°` marker never matched versioned model ids or "Claude X" display names (normalizer now strips prefix and date suffix); a missing `declared_at` was misreported as a stale budget (now an honest "corrupt file" deny). Plus two hot-path efficiency cuts ‚Äî the Stop hook no longer re-walks the whole transcript history every turn (sentinel-only records are skipped once satisfied) and `events(event, ts)` gets an index for the `[XF]` statusline query ‚Äî and doc drift fixes (gate matcher is `Agent|Task|Workflow` everywhere; benchmarks README now says 4 shapes / 24 sessions). Methodological note: on findings whose premise lay outside the shown snippet, Codex answered `uncertain` where Gemini said `refuted` ‚Äî same substance, better-calibrated label; both lanes confirmed every mechanism, none refuted a finding.
- **1.10.9 ‚Äî cost checkpoint: an expensive task is your call, not a silent spend.** When a task genuinely needs the top model (axis 2) *and* is expensive, you decide whether to spend the tokens ‚Äî the plugin no longer just burns them. For **delegated** work it's enforced: if the pre-budget's `expected-output` exceeds a ceiling (default 50k, `FD_COST_CEILING` or `cost-checkpoint.json` to tune), the gate returns `ask` instead of `allow`, surfacing the estimate and your remaining weekly quota ‚Äî approve and it proceeds, or reopen the budget with `--cost-ack` to stop re-asking on that task. The statusline now bridges your live quota to the gate (`quota.json`), so when the weekly limit runs low the ceiling drops automatically (a mid-size task becomes worth confirming). For **inline** work ‚Äî which no hook can intercept ‚Äî the kernel instructs the top model to present the choice itself: estimate, % of limit left, why the top model is needed, and the alternatives (split / cheap-executor+verify / defer to reset). It's the interactive, context-aware cousin of a hard dollar cap: not a blind stop, an informed decision that respects the rate limit as *your* scarce resource. Fail-open throughout.
- **1.10.8 ‚Äî the top model owns "done".** Before delegating, the kernel now requires a **verifiable done** (a command that passes / an observable behavior / an enumerable checklist) plus a **stop condition** with a failure cap ‚Äî and if done isn't verifiable, you don't delegate until it is. Executors never self-assess completion. This is the up-front twin of the verification ladder (which is *how* you check; this is *what* counts as finished) and closes the one gap peers like [helm](https://github.com/Jacknelson6/helm) and the 10-80-10 routing writeups all point at. It's the plan-half of "plan big, execute small": a cheap executor self-declaring success is exactly the failure this prevents.
- **1.10.7 ‚Äî the two silent things now show up in `report`.** (1) **read-dedup savings are measured, not silent:** every suppressed re-read logs a `read_dedup` event with the tokens saved (‚âà chars/4), and `report` sums them ‚Äî so enabling the opt-in hook produces a real number to validate against, not a vibe. (2) **Yield analysis (token per commit):** at `SessionEnd`, if the cwd is a git repo, the summary logs how many commits landed during the session window, and `report` prints output-tokens-per-commit. It measures *return* ‚Äî cost tells you what you spent, this tells you whether the spend produced kept work. Strictly a **diagnostic alarm, never a target**: planning/debug/review sessions legitimately don't commit, so a high (or zero) ratio flags where to look, it never condemns.
- **1.10.6 ‚Äî cross-family verifier is now data-driven by task type.** Rung 4 (the cross-family check) gains a `--type` slug that the telemetry logs, and `report` breaks the cross-family hit-rate down **per task type**. This answers *which task types cross-family actually catches errors on* empirically, instead of asserting model-family strengths (a claim that decays every release). Seed candidates in the skill ‚Äî `cross-lingua` (IT‚ÜîEN deliverables: a different family's multilingual training is genuinely uncorrelated), `security-review`, `spec-compliance` ‚Äî are a starting hypothesis confirmed or killed by the logged data (same `N‚â•10` discipline as the per-type budget density table), never doctrine. It stays a **skill-level suggestion**, not an auto-hook: auto-firing on a fuzzy "is this type X?" classification would burn free-tier quota.
- **1.10.5 ‚Äî lossless read-dedup (opt-in) + token-reduction stance.** New `read-dedup.py` PostToolUse hook: on a **re-read** of a file already seen this session it replaces Read's output with just the diff (or an "unchanged" marker), cutting the biggest agentic token sink ‚Äî re-sending file content already in context. **Lossless and recoverable:** it only touches large files, skips partial (offset/limit) reads, and after a dedup the *next* read of that file passes through in full ‚Äî so the model always has a one-read path back to full content even after a compact. **Opt-in by design** (not wired into the default hooks ‚Äî colleagues on the marketplace build don't inherit it until you validate it): enable with `export FD_READ_DEDUP=1` or `touch ~/.claude/fable-director/read-dedup.on`, then add the hook (see [Token reduction](#-token-reduction-lossless-only)). The routing engine optimizes **cost per token**; this is the first lever that cuts the **token count** itself ‚Äî and by policy it stays **lossless-only** (RAG-style lossy chunking is now a documented anti-pattern: ‚àí90% tokens bought with a correctness risk is the exact Goodhart failure the kernel forbids).
- **1.10.4 ‚Äî gate denials become telemetry.** Every deny from the pre-delegation gate now logs a `gate_deny` event (kind `no_budget` / `stale_budget` / `flagged`, plus tool and declared model) and `report` prints the breakdown. Without it, post-hoc analysis couldn't distinguish "never attempted delegation" from "denied and fell back inline" ‚Äî the exact blind spot hit while reading the shape-04 bench. Also fixed a systematically ambiguous shape-04 fixture: a quality-negative phrase ("pannello che si stacca") both arms read as a safety hazard against a ground truth of NO ‚Äî safety-precision from earlier runs is not comparable (see [benchmarks/README](benchmarks/README.md)).
- **1.10.2 ‚Äî cross-family lanes verified live.** Default Gemini model for new configs is `gemini-2.5-flash` (verified with a real call: `gemini-3-flash` doesn't exist on the AI Studio free tier, `gemini-flash-latest` answers 503 under load). Both lanes tested end-to-end ‚Äî Gemini free API and Codex CLI (ChatGPT login) each returned a correct adversarial verdict. **When they fire and how to invoke them yourself: see [Cross-family verifier](#-cross-family-verifier--when-and-how) below.**
- **1.10.1 ‚Äî `[DLG]` shows effective tokens, not declared calls.** The segment now reads the session transcript **incrementally** (a state file keeps the byte offset ‚Äî each refresh parses only new lines, never a rescan) and shows **output tokens per effective model** of subagent work: `[DLG SONNET-5 41k HAIKU-4-5 3k]` (`‚â°` = subagents inheriting the main-loop model). Immune to the quiet-fallback blind spot and it weighs work, not call counts. Where the transcript isn't exposed it degrades to the gate's declared-call registry, marked `‚âà` ‚Äî the two modes are visually distinct by design. Also new: **`[XF]` cross-family segment** ‚Äî `GEMINI‚ñ≤` while a `cross-verify.py` call is in flight, `CODEX√ó2` for today's calls (local telemetry; free tiers expose no quota API, so this is presence, not remaining quota).
- **1.10.0 ‚Äî `[DLG]` statusline segment (declared calls).** The pre-delegation gate keeps a per-session registry of delegations counted by declared model. The registry dies at SessionEnd (48h orphan cleanup for crashed sessions).
- **Benchmark task shape 04 ‚Äî where governance actually bites.** 40 synthetic customer reviews requiring per-item semantic judgment (closed-vocabulary sentiment/theme + safety-defect flagging, 6 planted defects with ground truth outside the task's view). Unlike shapes 01‚Äì03 (script-parity), this one can trigger real delegation ‚Üí the gate and Stop hook finally get exercised in the `on` arm. `aggregate.py` now also reports **accuracy per arm** (sentiment/theme accuracy, safety recall/precision): savings only count at verified-equal quality ‚Äî safety recall lost in one arm gets reported, not hidden.

Warn only, never auto-update ‚Äî a nested CLI call in a hook is slow, races the plugin registry, and the current session would stay on the old version anyway.

## 1.7.0
